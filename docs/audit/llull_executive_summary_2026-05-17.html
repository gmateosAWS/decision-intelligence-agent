Método de trabajo · evidencia operativa

Las decisiones arquitectónicas se toman entre Claude (asistente arquitecto) y el equipo humano antes de redactar cualquier prompt para Claude Code. Cada item completa un ciclo: especificación en chat de arquitectura → prompt detallado a Claude Code → plan review obligatorio (capturando riesgos concretos antes de tocar código) → implementación → tests → documentación → PR a main.

La disciplina del Plan review se introdujo en el sprint actual tras observar que Claude Code tendía a ejecutar sin cuestionar y reproducir bugs sutiles. Aplicada en los últimos dos PRs (10.2 A/B testing y 5.9 GroundedTokens), ha capturado 6 y 6 riesgos concretos respectivamente antes de tocar código, incluyendo bugs sutiles de integración con LangGraph y try/except amplios del planner. La adopción de esta disciplina es ya una fortaleza confirmada.
