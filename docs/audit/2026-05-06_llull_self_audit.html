Critical findings · 6

P0 = blocker · P1 = important · P2 = tactical
6.1 No executable CI pipeline P0 · Codebase #14, #24
Inventory item 11.1 Pipeline CI is in I1 but no .github/workflows/ exists. 1.300 LOC of tests across 14 files cannot be exercised automatically. pre-commit hooks protect local commits, not pushes.
6.2 Spec loaded at module-import time in config/settings.py P1 · Codebase #11
config/settings.py:14-37 calls get_spec() at module load. A YAML parse error or DB unreachable crashes the import cascade before logging is configured. The recent commit 5d2adf5 fixed the same antipattern in the planner but skipped settings.py.
6.3 pytest missing from requirements-dev.txt P1 FIXED · Codebase #8, #13
requirements-dev.txt lists only black, ruff and pre-commit. 14 test files require pytest via convention. Without manual install, tests cannot run. — Fixed: added pytest~=8.0 and pytest-cov~=5.0 (audit-P03).
6.4 Inconsistent declared Python target version P2 · Codebase #8, #16
.python-version says 3.12; pyproject.toml declares target-version = ["py310"] for black. No CI matrix to validate.
6.5 CORS allows wildcard methods + headers with credentials P2 · Codebase #15 · raised to P0 on public exposure
api/main.py:84-89: allow_methods=["*"], allow_headers=["*"], allow_credentials=True is silently inconsistent (CORS spec disallows wildcard with credentials in browsers).
6.6 FAISS allow_dangerous_deserialization without comment P2 · Codebase #15
knowledge/retriever.py:130: allow_dangerous_deserialization=True without inline threat-model comment. Today low risk (locally generated indices); grows with multi-tenancy or remote indices.

Genuine strengths · 9

Invariants for future audits

Single coherent architecture
Four-node LangGraph pipeline is the sole runtime path. Zero zombie code, zero coexisting architectures.
8.2
Spec-driven design that is actually spec-driven
Spec consumed by every layer that should consume it. New domain = edit YAML, restart.
8.3
Type discipline above the size class
27 Pydantic / TypedDict refs in 8.372 LOC. Structured outputs at every LLM seam. 2 type:ignore total.
8.4
Dual-backend pattern consistent across modules
Postgres ↔ SQLite, pgvector ↔ FAISS — same pattern across memory, spec, knowledge, evaluation.
8.5
Observability built in, not bolted on
AgentObserver records per-node spans, latency, model, judge score. Dual JSONL + Postgres. LangSmith.
8.6

38 broad excepts, 27 marked noqa: BLE001 — declared intent, not accidental swallowing.
8.7
Active fix discipline visible in commits

8.8
Documentation that matches the code

8.9
Tests that test behavior, not mocks
